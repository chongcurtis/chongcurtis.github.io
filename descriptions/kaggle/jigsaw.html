Premise

Every day thousands of questions are asked on Quora, but not all questions were asked with positive intentions. Quora calls these questions insincere. Intrigued by the issue, my teammate Kevin Shen and I joined the competition to predict if a submitted question had violated Quora’s terms of service. The cross validation for this competition was an f1 score between these two classes: sincere and insincere problems.
Like all Kernel Competitions, we only had 2 hours to load data, preprocess, train, and predict classes without the use of external data. Yikes, this killed any opportunity for large robust ensembles and heavy feature engineering. Since the only provided variable were the questions, we began with heavy EDA on the words used,

Exploratory Data Analysis

By now have a fairly standard method for text data. We’d run a script for token correlations, the most common n-grams for each class, an analysis on sentence structure using NLTK, and of course, just reading the questions in the dataset. Surprise surprise, most of the flagged questions were blatantly racist or inherently stupid.

Preprocessing

Next, we followed the standard procedure of cleaning the data and generating a custom dictionary for misspellings, importing the text embeddings etc. Unfortunately, the FastText .bin file was not provided so we couldn’t generate out of vocabulary embeddings (Since all processing would have to be completed in 2 hours, re-generating embeddings weren’t an option)

Approach

We began by spot checking with a variety of classical algorithms. However, algorithms such as the SVM, Random Forests, and logistics regression could only pull an F1 score of 0.4-0.5. So we started finding the embeddings that had the best coverage for the dataset and we settled on an average of the GLove and Paragram embeddings (Paragram had more coverage but, Glove usually performed better. Long story short, we settled on a bidirectional GRU with Capsules and an attention layer as our final model. We also included pretty standard meta-features such as the number of capitalized letters, sentence length, and the number of different symbols in each question.

One surprising thing was that using the first word of the question as a one-hot encoded feature didn’t work too well. Another thing that amazed us was when we removed the top 100 most common words in each class and replaced it with a neutral word either (bacon, eggs, or ham), our f1 score increased by 0.02. We had this idea because we wanted the network to understand the underlying semantics of how insincere questions were semantically worded instead of blindly looking for the right keywords. Granted, we added another 2 features that stated the number of common insincere/sincere words that were found in each question (to increase training time).

All in all, this led to a pretty stable model and we ended with an f1 score of 0.685!
